The file must have YAML frontmatter (between triple-dash delimiters) containing:
  - description: "Run rms code review — prompts for scope (local diff or PR) if not specified"
  - argument-hint: "[local | pr &lt;pr-number&gt;] [--focus &lt;area&gt;]"
  - subtask: true

The body (after closing frontmatter) must be exactly:
  !node dist/index.js review $ARGUMENTS

Reference: opencode-review-local.md has the same structure — description, argument-hint, subtask: true, then !node dist/index.js review-local $ARGUMENTS. This template is identical except the command is `review` instead of `review-local`.

**Why this works in OpenCode:**
- `subtask: true` runs the command in a fresh isolated agent context
- The !command injects CLI stdout into the subtask agent's response
- When called with no args, the CLI outputs the scope prompt — the subtask agent presents it to the user
- When called with `local` or `pr N`, the CLI runs the full pipeline

**Do not add prose instructions** — the !command pattern handles everything through stdout injection. Keep it minimal like the other OpenCode templates.
