  <action>
  Create `src/diff-preprocessor.ts`.

  The preprocessor takes a raw `git diff` string and returns a cleaned string with lock files, binary content, and node_modules diffs removed. It also returns stats about what was stripped.

  **Exact strip rules (implement ALL):**

  1. **Lock file diffs** — strip entire file diff section when the diff path matches any of:
     - `package-lock.json`
     - `yarn.lock`
     - `pnpm-lock.yaml`
     - `*.lock` (any file ending in `.lock`)
     - `Gemfile.lock`
     - `Cargo.lock`
     - `poetry.lock`
     - `composer.lock`

  2. **node_modules diffs** — strip entire file diff section when path contains `node_modules/`

  3. **Binary file diffs** — strip entire file diff section when the diff block contains the git binary marker:
     - Line matching: `^Binary files .* differ$`
     - Line matching: `^GIT binary patch`

  4. **Generated/minified files** — strip entire file diff section when path matches:
     - `*.min.js`
     - `*.min.css`
     - `dist/` path prefix
     - `build/` path prefix

  **Diff block detection:** A git diff file section starts with `diff --git a/... b/...` and ends at the next such line (or EOF). Strip the whole block for any matching file.

  **Interface:**
  ```typescript
  export interface DiffStats {
    originalLines: number;
    strippedFiles: string[];
    remainingLines: number;
  }

  export function preprocessDiff(rawDiff: string): { diff: string; stats: DiffStats };
  ```

  **Implementation approach:**
  - Split the raw diff into file sections at `diff --git` boundaries
  - For each section, extract the filename from the `diff --git a/{path} b/{path}` header line
  - Apply strip rules against the extracted path
  - Rejoin kept sections

  Create `src/diff-preprocessor.test.ts` using `node:test`. Tests MUST cover:
  1. `package-lock.json` section is stripped entirely — result does not contain `package-lock.json`
  2. `yarn.lock` section is stripped
  3. `node_modules/some-package/index.js` diff is stripped
  4. A regular `.ts` file diff passes through unchanged
  5. Binary file marker (`Binary files a/foo.png b/foo.png differ`) causes section to be stripped
  6. `stats.strippedFiles` contains the names of all stripped files
  7. Mixed diff with 2 lock files + 1 real file: result contains only the real file's diff
  8. Empty string input returns empty string with zero stats

  Run: `npx tsx --test src/diff-preprocessor.test.ts`
  </action>

  **Function 1: `getLocalDiff`**
  ```typescript
  import simpleGit from 'simple-git';
  import { preprocessDiff, DiffStats } from './diff-preprocessor.js';

  export interface LocalDiffResult {
    diff: string;           // preprocessed diff content
    stats: DiffStats;       // what was stripped
    hasStagedChanges: boolean;
    hasUnstagedChanges: boolean;
  }

  export async function getLocalDiff(projectRoot: string): Promise<LocalDiffResult>
  ```

  Implementation:
  - Create a `simpleGit(projectRoot)` instance
  - Get staged diff: `git.diff(['--cached'])` → string
  - Get unstaged diff: `git.diff([])` → string
  - Combine: `rawDiff = stagedDiff + unstagedDiff`
  - Call `preprocessDiff(rawDiff)` to strip noise
  - Return `{ diff, stats, hasStagedChanges: stagedDiff.length > 0, hasUnstagedChanges: unstagedDiff.length > 0 }`

  **Function 2: `writeInputFile`**
  ```typescript
  export interface WriteInputOptions {
    sessionDir: string;   // absolute path, e.g. .reviews/2026-04-05-local/

    scope: 'local-diff' | 'pr-diff';
    focus?: string;       // optional focus area

  }

  export async function writeInputFile(opts: WriteInputOptions): Promise<string>
  ```

  **Function 3: `parseReviewerOutput`**
  ```typescript
  export interface ParsedReviewerOutput {
    findings: Array<Omit<Finding, 'id'>>;  // id assigned later by orchestrator
    dimensionsCovered: Dimension[];
    dimensionsWithFindings: Dimension[];
    rawContent: string;
  }

  export async function parseReviewerOutput(reviewerMdPath: string): Promise<ParsedReviewerOutput>
  ```

  Parse REVIEWER.md to extract structured findings. The reviewer writes `<finding>` blocks with YAML-like key:value lines inside. Parse rules:
  - Read the file at `reviewerMdPath`
  - Find all `<finding>...</finding>` blocks (multiline regex: `/<finding>([\s\S]*?)<\/finding>/g`)
  - For each block, parse key:value lines:
    - `severity: {value}` → severity
    - `file: {value}` → file
    - `line: {value}` → line
    - `dimension: {value}` → dimension
    - `explanation: {value}` → explanation (first line; multi-line handled below)
    - `suggestion: {value}` → suggestion (first line; multi-line handled below)
  - For multi-line `explanation` and `suggestion`: capture all lines until the next `key:` or `</finding>`
  - Validate each parsed finding against `FindingSchema.omit({ id: true })` — skip and warn if invalid
  - Track `dimensionsCovered`: extract all `## {DIMENSION}` section headers present in the file
  - Track `dimensionsWithFindings`: dimensions that have at least one parsed `<finding>` block
