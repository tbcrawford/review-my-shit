<objective>
Rewrite the three OpenCode model picker command templates to use agent-presented selection instead of TTY-dependent CLI TUI.

**Problem:** The current templates use `subtask: true` + `!rms reviewer` which runs the @inquirer/prompts TUI in a non-TTY subprocess — immediately producing "Cancelled." because stdin is not interactive.

**Solution:** Remove `subtask: true`. Use `tools: question: true` so the agent can present the tier choices via the conversation interface. After the user selects, the agent calls `!rms settings --reviewer <spec>` to apply.

Purpose: Make `/rms-reviewer`, `/rms-validator`, `/rms-writer` actually work in OpenCode.
Output: 3 updated template files in `src/templates/`.
</objective>
