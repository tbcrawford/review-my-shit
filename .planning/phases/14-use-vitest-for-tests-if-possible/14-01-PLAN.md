<task type="auto">
  <name>Task 2: Create vitest.config.ts</name>
  <files>vitest.config.ts</files>
  <read_first>tsconfig.json</read_first>
  <action>
Create `vitest.config.ts` in the project root with this exact content:

```typescript
import { defineConfig } from 'vitest/config'

export default defineConfig({
  test: {
    environment: 'node',
    include: ['src/**/*.test.ts'],
    globals: false,
  },
})
```

Explanation of choices:
- `environment: 'node'` — tests use Node.js APIs (fs, os, path, tmpdir). No browser environment needed.
- `include: ['src/**/*.test.ts']` — matches all 13 existing test files.
- `globals: false` — use explicit imports (`import { test, describe, expect } from 'vitest'`) rather than globals. This avoids needing `types: ["vitest/globals"]` in tsconfig and stays compatible with NodeNext module resolution.

Do NOT add `coverage`, `reporters`, or `pool` config — keep it minimal.
  </action>
  <acceptance_criteria>
    - vitest.config.ts exists at project root
    - File contains `environment: 'node'`
    - File contains `include: ['src/**/*.test.ts']`
    - File contains `globals: false`
    - File imports `defineConfig` from `'vitest/config'`
    - `bun run typecheck` still exits 0 (tsconfig.json unchanged)
  </acceptance_criteria>
  <verify>
    <automated>node -e "const fs = require('fs'); const c = fs.readFileSync('vitest.config.ts','utf8'); if(!c.includes(\"environment: 'node'\")) throw new Error('missing environment: node'); if(!c.includes(\"globals: false\")) throw new Error('missing globals: false'); console.log('vitest.config.ts OK')"</automated>
  </verify>
  <done>vitest.config.ts exists with correct node environment and explicit-imports mode</done>
