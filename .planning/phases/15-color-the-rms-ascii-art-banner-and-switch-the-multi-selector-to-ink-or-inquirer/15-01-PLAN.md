Step 2 — Update src/setup.ts:

Replace the plain BANNER_STRING with a chalk-colored version. Use this exact color scheme:
- Box border characters (`┌`, `─`, `┐`, `│`, `└`, `┘`) → `chalk.cyan`
- "rms" → `chalk.bold.white`
- "review-my-shit" → `chalk.dim.white` (or `chalk.gray`)
- version string like "v0.3.0" → `chalk.yellow`
- " — " separator → `chalk.dim`

  b) Keep it as `export const BANNER_STRING` but accept that in test environments where chalk is disabled (level=0), it will be plain text — this is fine because chalk auto-detects color support.

Step 3 — Update src/setup.test.ts:

The existing test checks `BANNER_STRING.includes('rms')` and `BANNER_STRING.includes('v0.3.0')`. In CI/test environments chalk may produce plain output (level=0) OR colored output. Update the test to strip ANSI escape codes before checking:

```typescript
import { describe, it, expect } from 'vitest';
import { resolveEditorsFromArgs, BANNER_STRING } from './setup.js';

// Strip ANSI escape codes for content testing
function stripAnsi(str: string): string {
  return str.replace(/\u001b\[[0-9;]*m/g, '');
}

describe('BANNER_STRING', () => {
  it('banner contains "rms" and version after stripping ANSI', () => {
    const plain = stripAnsi(BANNER_STRING);
    expect(plain.includes('rms')).toBeTruthy();
    expect(plain.includes('v0.3.0')).toBeTruthy();
  });
});
```
