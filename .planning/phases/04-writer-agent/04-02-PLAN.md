**Plan:** 04-02

## Goal

Wire `runWriter` into `src/index.ts` as Step 6, pass all required parameters, update the CLI output to show the final REPORT.md path, and add an integration test that verifies the full pipeline (reviewer → validator → writer) produces a complete audit trail.

## Requirements Covered

- REPT-04: `.reviews/<session>/` contains REVIEWER.md, VALIDATOR.md, and REPORT.md
- PIPE-05: Writer wired into the full pipeline

## Success Criteria

This plan is done when:
1. `rms review-local` produces REPORT.md in `.reviews/<session>/` alongside REVIEWER.md and VALIDATOR.md
2. CLI output shows path to REPORT.md
3. Integration test verifies: reviewer finding count ≥ 1, validator verdict count matches, REPORT.md exists and contains all finding IDs
4. TypeScript compiles clean (`npm run build`)
5. All tests pass (`npm test`)

## Tasks

### Task 1: Wire `runWriter` into `src/index.ts`

In the `review-local` command action, after `runValidator`:

1. Read INPUT.md content (already written at `inputMdPath`) for passing to `runWriter`
2. Resolve `modelId` string — update `resolveModel()` or extract from env vars at call site:
   ```typescript
   const modelId = process.env['AI_SDK_MODEL'] ?? 'gpt-4o';
   ```
3. Call `runWriter`:
   ```typescript
   const writerResult = await runWriter({
     session,
     findings: result.findings,      // Finding[] with IDs from runReviewer
     verdicts: validatorResult.verdicts,
     validatorRawContent: validatorResult.rawContent,
     inputMdContent: await readFile(inputMdPath, 'utf8'),
     dimensionsCovered: result.dimensionsCovered,
     modelId,
   });
   ```
4. Check that `runReviewer` result exposes `findings` and `dimensionsCovered` — if not, update `ReviewerResult` in `reviewer.ts` to include them (they're returned by `parseReviewerOutput` already)
5. Check that `runValidator` result exposes `rawContent` — if not, update `ValidatorResult` in `validator.ts` to include it

### Task 2: Update CLI output

Replace the current `(Writer agent will be added in Phase 4)` placeholder with:
```
  REPORT.md: .reviews/{reviewId}/REPORT.md
```

And add finding counts to the summary:
```
Review complete:
  Findings: {reviewer findings}
  Verdicts: {verdictCount} ({challenged} challenged, {escalated} escalated)
  Counter-findings surfaced: {counterFindingCount}
  Total findings in report: {findingCount}

Audit trail:
  INPUT.md:     .reviews/{reviewId}/INPUT.md
  REVIEWER.md:  .reviews/{reviewId}/REVIEWER.md
  VALIDATOR.md: .reviews/{reviewId}/VALIDATOR.md
  REPORT.md:    .reviews/{reviewId}/REPORT.md
```

### Task 3: Add integration test (pipeline round-trip)

In a new `src/integration.test.ts` or appended to an existing integration test file:

Test: full mock pipeline produces complete audit trail
- Create a synthetic reviewer mock output (3+ findings, mix of severities)
- Create a synthetic validator mock output (1 confirmed, 1 challenged with counter-finding, 1 escalated)
- Run `runWriter` with the parsed structured data
- Assert: REPORT.md exists in temp session dir
- Assert: All 3 reviewer finding IDs appear in REPORT.md
- Assert: Counter-finding ID appears in REPORT.md
- Assert: Escalated finding has bumped severity in REPORT.md
- Assert: `findingCount` in WriterResult matches total entries

### Task 4: Verify `npm test` passes for all suites

Run the full test suite. Fix any broken tests caused by interface changes to `ReviewerResult` or `ValidatorResult`.
