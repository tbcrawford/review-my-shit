RmsConfig shape (src/schemas.ts):
```typescript
interface AgentModelSpec {
  model: string;
  variant?: 'high_thinking' | 'no_thinking';
}
interface EditorAgentConfig {
  reviewer: AgentModelSpec;
  validator: AgentModelSpec;
  writer: AgentModelSpec;
}
interface RmsConfig {
  opencode: EditorAgentConfig;
  cursor: EditorAgentConfig;
}
```

config.ts exports:
```typescript
loadRmsConfig(configPath?: string): Promise<RmsConfig | null>
saveRmsConfig(config: RmsConfig, configPath?: string): Promise<void>
resolveAgentModel(spec: AgentModelSpec): Promise<LanguageModel>
DEFAULT_RMS_CONFIG: RmsConfig  // has .opencode and .cursor sections
```

  <name>Task 1: Update resolveModels() and the settings command</name>
  <files>src/index.ts</files>
  <action>
    **resolveModels():**
    - Remove env-var fallback branch (no longer needed — copilot is the only provider)
    - Read `config.opencode.reviewer`, `config.opencode.validator`, `config.opencode.writer`
    - Call `resolveAgentModel(spec)` for each (new single-arg signature — no provider)
    - If no config file, fall back to `DEFAULT_RMS_CONFIG.opencode.*`
    - Return `{ reviewerModel, validatorModel, writerModelId }` where writerModelId is `config.opencode.writer.model`

    **parseSpec() helper (local to settings command):**
    Old: parsed "provider:model" or "github-copilot/model-id"
    New: parses "model" or "model:variant"
    ```
    "github-copilot/claude-opus-4.6:high_thinking"  → { model: "github-copilot/claude-opus-4.6", variant: "high_thinking" }
    "github-copilot/claude-opus-4.6"                → { model: "github-copilot/claude-opus-4.6" }
    "claude-4.6-opus-high-thinking"                  → { model: "claude-4.6-opus-high-thinking" }
    "claude-4.6-opus-high-thinking:no_thinking"      → { model: "claude-4.6-opus-high-thinking", variant: "no_thinking" }
    ```
    Split on last `:` only if the right-hand side is exactly `high_thinking` or `no_thinking`. Otherwise treat entire string as model ID.

    **settings command:**
    - Add `--opencode` and `--cursor` flags (boolean options, no args)
    - If `--reviewer`/`--validator`/`--writer` flags provided:
      - If `--opencode` only: update only `config.opencode.*`
      - If `--cursor` only: update only `config.cursor.*`
      - If both flags or neither: update both sections
    - Construct updated RmsConfig by spreading existing config and replacing the targeted section(s)
    - Display logic (no flags): show both `opencode` and `cursor` sections side by side:
      ```
      Config: ~/.config/rms/config.json

        opencode:
          reviewer   github-copilot/claude-opus-4.6 [high_thinking]
          validator  github-copilot/gpt-5.4 [high_thinking]
          writer     github-copilot/claude-haiku-4.5 [no_thinking]

        cursor:
          reviewer   claude-4.6-opus-high-thinking
          validator  gpt-5.4-high
          writer     gpt-5.4-mini-none
      ```

    **reviewer / validator / writer subcommands:**
    These pick the model for a specific agent. Update to:
    - Accept optional positional `<spec>` argument (e.g. `rms reviewer "github-copilot/claude-opus-4.6:high_thinking"`)
    - If spec provided: parse via parseSpec(), save to BOTH editor sections (reviewer field only)
    - If no spec: launch runModelPicker (existing TTY picker), save result to both editor sections
    - Update the save logic: spread the existing config, replace `.opencode.{agent}` and `.cursor.{agent}` with the new spec (variant only applies meaningfully to opencode, but store it in both for simplicity)
    - Keep existing ExitPromptError catch and "Cancelled." message
    - Update console.log confirmation to show the model + variant:
      `✓ Reviewer set to github-copilot/claude-opus-4.6 [high_thinking]`

    **AgentModelSpec import:** Update import from `./schemas.js` to remove `AgentModelSpec` (it's no longer used directly in index.ts for the fallback spec construction — remove the env-var fallback). Keep the type import if still needed for parseSpec return type.
