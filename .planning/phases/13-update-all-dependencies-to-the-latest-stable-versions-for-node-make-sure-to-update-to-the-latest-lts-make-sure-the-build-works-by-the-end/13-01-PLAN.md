<task type="auto">
  <name>Task 2: Fix tsconfig.json for TypeScript 6 compatibility</name>
  <files>tsconfig.json</files>
  <read_first>
    - tsconfig.json — read current state before modifying
  </read_first>
  <action>
Add `"types": ["node"]` to the compilerOptions in tsconfig.json.

This is the ONLY required change for TypeScript 6 compatibility in this project. TypeScript 6 no longer auto-discovers `@types` packages — without this, the build will fail with errors like `Cannot find name 'process'` and `Cannot find module 'fs'`.

The resulting compilerOptions must include:
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "outDir": "dist",
    "rootDir": "src",
    "strict": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "types": ["node"]
  }
}
```

Do NOT change any other tsconfig options — all existing settings are already correct for TypeScript 6:
- `rootDir: "src"` — already explicit, avoids TS6 rootDir default change
- `strict: true` — already set, matches TS6 new default
- `moduleResolution: "NodeNext"` — already set, not deprecated in TS6
  </action>
  <verify>
    <automated>node -e "const c=JSON.parse(require('fs').readFileSync('tsconfig.json','utf8')); console.log(JSON.stringify(c.compilerOptions.types))" | grep -F '["node"]'</automated>
  </verify>
  <acceptance_criteria>

  </acceptance_criteria>
